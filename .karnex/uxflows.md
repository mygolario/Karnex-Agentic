## The Core Design Philosophy

The founder should never think about which agent to use, which layer to navigate to, or what to type. They should only ever think about what they want to accomplish next.

## Consolidated User Flows

### Flow 1: The Magic Onboarding (< 10 minutes)

```
Step 1: Pain Input (60s)
   └─ Single high-impact text area: "What pain kept you up last night?"
   └─ Dynamic placeholder examples showing real-world founder frustrations.
   └─ CTA: "Transform this into a product"

Step 2: Pain-to-Product score cards (15s)
   ├─ Pain-to-Product Transformer runs in background.
   ├─ Animated loading: "Scanning market gaps...", "Scoring buildability..."
   └─ 3 hypothesis cards appear side-by-side with overall scores.
   └─ Founder selects one hypothesis.
   └─ CTA: "Crystallize this idea"

Step 3: Product Brief & ICP Personas (2min)
   ├─ Idea Crystallizer + ICP Definer run in parallel in the background.
   └─ Beautiful one-page Product Brief reveals:
       ├─ Suggested Name & Tagline
       ├─ Prioritized Feature Set (Must-Haves)
       └─ 3 Ideal Customer Profile (ICP) Personas (e.g. Mia from Portland)
   └─ CTA: "Generate my roadmap"

Step 4: Roadmap Generation & Sprints (30s)
   ├─ 90-day roadmap generated by the War Room system.
   ├─ Timeline animation displays: Phase 1 (Validation) -> Phase 2 (Building) -> Phase 3 (Launch)
   ├─ "Sprint 1 is loaded. Your co-founder is ready."
   └─ CTA: "Open my workspace"
```

---

### Flow 2: Daily Standup (Journey Page HQ)

```
1. TRIGGER
   └─ On loading the Home (Journey) page, if standup is not done for the day:
      └─ Display "TODAY'S STANDUP" widget at the top of the main panel.

2. STANDUP INPUT
   └─ Form asks: "What did you ship yesterday? (or skip if done)"
   └─ Text area for input.
   └─ Button: "Finish Standup" or "Skip"

3. COACHING FEEDBACK
   ├─ Daily Standup agent evaluates input in background (3-5s).
   ├─ Standup form is dismissed (with fade-out animation).
   └─ Replaced by "Karnex Says" coaching insight panel:
       └─ Calculates momentum score delta (e.g. +8).
       └─ Outputs contextual advice based on historical context and current tasks.
```

---

### Flow 3: The "Let Karnex" Action (Slide-Over Console)

```
1. TRIGGER
   └─ Founder clicks "Let Karnex" on one of the maximum 3 daily task cards.

2. SLIDE-OVER OPENS (No Prompting Required)
   ├─ System reads `tasks.agent_config` (JSONB) pre-populated during sprint planning.
   ├─ Renders a clean slide-over panel summarizing in plain English:
       └─ "Here is what the agent will do next:"
       └─ Bullet points of actions, targets, and expected outputs.
   └─ CTA Button: "Go — let Karnex handle this"

3. EXECUTION PIPELINE
   ├─ Task card shows real-time progress bar + changing step labels.
   ├─ Supabase Realtime streams updates from the Python service.
   └─ Typical steps: Queued -> Decomposing specs -> Spawning agent -> Running lint -> Success.

4. COMPLETION
   ├─ Task is automatically marked completed in the DB.
   ├─ Generated output (JSONB) attaches directly to the task card for inspection.
   └─ Artifact is automatically saved to the founder's Vault in the background.
```

## Screen Layout Specifications

### 1. Home (The Journey Page)

*   **Header Section**: "Good morning, Sarah. Day 47 of 90." + Progress bar showing phase progress (e.g. "Phase 2: Building (52% complete)").
*   **Standup Widget**: Interactive textarea at the top. Slides up/out on submission.
*   **Today's Tasks Grid**: Exactly 3 task cards (never more).
    *   Each card features a title, brief description, category badge, and a `Let Karnex` action button.
*   **Metrics Sidebar / Row**:
    *   **Momentum Widget**: Radial/bar chart displaying score (0-100).
    *   **Streak Widget**: Active day streak counter (e.g. "12 days").
    *   **Credits Widget**: Task usage meter (e.g. "342/500 credits").
*   **Coaching Insight Box**: "Karnex Says" feedback bubble at the bottom.

---

### 2. Studio (CTO-Over-Shoulder Workspace)

Default view (90% of users):
*   **Left Column: Karnex Says (CTO Chat)**:
    *   Clean dialogue interface showing Karnex's recommendations, active building logs, and input field.
    *   **"Go — build this"** prominent button.
    *   **Advanced Mode Toggle**: Reveals code tree, schema viewer, and configuration panels.
*   **Right Column: Live Preview & Status**:
    *   Embedded iframe pointing to the development branch preview (e.g., `folio-preview.vercel.app/invoicing`).
    *   Active progress timeline overlay during building:
        *   `[x]` Supabase table created
        *   `[ ]` API routes scaffolded
        *   `[ ]` Generating UI components...
*   **Workspace Action Buttons**:
    *   **Deploy**: Pushes the branch to production.
    *   **Review Code**: Opens diff inspector in Advanced Mode.
    *   **Share Preview**: Copies public preview link.

---

### 3. Integrations Hub

*   **Level 1: Connect Cards**: Grid of priority connectors (GitHub, Gmail, Vercel, Resend, Zapier, etc.). Displays value-driven subtitles: *"Connect GitHub: Builder Agent pushes code directly to your repo."*
*   **Level 2: Automate Recipes**: Toggle-able automation workflows:
    *   *"When Builder finishes -> auto-create GitHub PR -> email review"*
    *   *"When Momentum Score < 40 -> trigger Accountability Agent"*
    *   *"When weekly sprint ends -> auto-trigger Weekly Debrief"*
*   **Level 3: Status Panel**: List of active credentials, token health indicators, remaining rate limits, and reconnect buttons.

---

### 4. Vault (All Outputs & Assets)

*   **Workspace Assets**: Product brief, defined ICP personas, competitive matrix, and decision journals.
*   **Agent Deliverables**: Directory list of code bundles, research PDFs, email outreach lists, and copy documents.
*   **Interaction**: Click to preview, download, or copy to clipboard.

---

### 5. Settings Hub

Contains a tabbed container with three sub-views:
*   **Profile Tab**: Display name, timezone, weekly availability slider (1-80 hrs), and communication tone.
*   **Billing Tab**: Gated subscription management.
    *   *Starter ($29/mo)*: 100 tasks, manual execution.
    *   *Builder ($79/mo)*: 500 tasks, autonomous execution, GitHub/Gmail live.
    *   *Founder ($149/mo)*: Unlimited, autonomous background execution.
    *   *Studio ($299/mo)*: Everything + multi-project + team seats.
    *   *Upgrade Trigger*: "I ran out of tasks" or "I want this to run automatically."
*   **Advanced Tab**: Agent Registry showing active agent configurations, task quotas, and execution logs.

## Agent Execution State Machine

```
              ┌────────────┐
              │    IDLE    │
              └─────┬──────┘
                    │ User clicks "Let Karnex" or "Go build this"
                    ▼
              ┌────────────┐
              │   QUEUED   │ ← Quota/credit check passes
              └─────┬──────┘
                    │ Process picked up by services/
                    ▼
              ┌────────────┐
              │  RUNNING   │ ← Realtime status updates streamed
              └─────┬──────┘
                    ├────────────────────────┐
                    ▼                        ▼
              ┌────────────┐           ┌────────────┐
              │  SUCCESS   │           │   ERROR    │
              └─────┬──────┘           └─────┬──────┘
                    │ Auto-mark Done         │ Display retry CTA
                    ▼                        ▼
              ┌────────────┐           ┌────────────┐
              │ Vault Save │           │ Human Log  │
              └────────────┘           └────────────┘
```
