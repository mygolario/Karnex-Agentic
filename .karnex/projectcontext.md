# Karnex — Project Context

> **Single Source of Truth** — Any agent, developer, or stakeholder reading this file should fully understand what Karnex is, who it's for, and how it works.

---

## Executive Summary

Karnex is a multi-agent AI platform that serves as the AI operating system for solo founders and indie hackers — collapsing the gap between raw ambition and a launched product with real revenue. Unlike chatbots or productivity tools, Karnex deploys specialized AI agents organized across five architectural layers (Dream Engine, Architect, Executor Pack, Compass, Platform) that don't just advise but execute real work: validating ideas, building codebases, running outreach campaigns, coaching founders through setbacks, and tracking progress toward paying customers. Karnex is the co-founder that shows up every day, does the work, and never quits.

---

## Problem Statement

> *"I have this idea that keeps me up at night. I know it could work. But I'm one person. I can't build the product, validate the market, do customer outreach, figure out pricing, write landing page copy, set up payments, AND stay motivated — all at the same time, all alone. Every morning I open my laptop and stare at 47 open tabs, not knowing what to do first. I've read every blog post, watched every YouTube video. I don't need more advice. I need someone — something — that will actually do the work WITH me. A co-founder who never sleeps, never judges, and never quits."*

### The Pain in Detail

Solo founders face a **compounding execution gap**:

1. **No co-founder** — No one to divide labor with, no one to challenge assumptions, no one to hold them accountable.
2. **No mentor** — They make preventable mistakes because they don't know what they don't know.
3. **No team** — Every function (engineering, marketing, sales, finance, legal) falls on one person.
4. **No structure** — Without a forcing function, they drift between tasks, lose momentum, and eventually abandon the project.
5. **No feedback loops** — They operate in an echo chamber, unable to distinguish signal from noise.

The result: **95% of solo founder projects die not from bad ideas, but from execution collapse.**

## Solution & Positioning

### What Karnex Is

Karnex is the **AI co-founder stack** — a platform of specialized AI agents that execute the actual work of building a startup, organized into five layers that mirror the functions of a real founding team.

### What Karnex Is NOT

- **Not a chatbot.** You don't "chat" with Karnex. You command it, and it executes.
- **Not a productivity tool.** It doesn't help you organize tasks. It does the tasks.
- **Not a code assistant.** It includes a Builder Agent, but Karnex is much broader — it covers idea validation, outreach, coaching, financial modeling, and more.
- **Not a SaaS template.** It doesn't give you a boilerplate. It builds your specific product.

### Positioning Statement

> For solo founders who have the ambition but not the team, Karnex is the AI operating system that turns raw ideas into launched products with real revenue — by deploying specialized agents that execute every function of a founding team.

### Layer 1 — Dream Engine (Idea Intelligence)

**Purpose:** Transform raw pains, frustrations, and half-formed ideas into validated, structured product opportunities.

**Philosophy:** Most founders start with a solution, not a problem. Dream Engine forces problem-first thinking by extracting the pain, mapping it to markets, and crystallizing it into a product concept worth building.

| Agent | Role |
|-------|------|
| Pain-to-Product Transformer | Takes raw frustration descriptions and outputs structured product hypotheses with problem/solution fit scores |
| Idea Crystallizer | Refines a raw idea into a structured product brief: name, tagline, value proposition, feature set, pricing hypothesis |
| Trend Radar | Monitors industry trends, emerging technologies, and market signals to contextualize and validate the idea |
| Competitive Landscape Agent | Maps competitors, identifies gaps, and positions the founder's idea in the market |
| ICP Definer | Defines the Ideal Customer Profile: demographics, psychographics, pain intensity, willingness to pay, and channels to reach them |

### Layer 2 — The Architect (Roadmap & Strategy)

**Purpose:** Build and continuously adapt the full founder journey from idea to revenue.

**Philosophy:** Solo founders don't fail because they lack a plan — they fail because their plan doesn't adapt. The Architect creates living roadmaps that evolve with every new data point.

| Agent | Role |
|-------|------|
| 90-Day War Room | Generates a complete 90-day roadmap: phases, milestones, weekly goals, and success criteria |
| Sprint Planner | Breaks the current phase into weekly sprints with specific, actionable tasks |
| Decision Journal | Logs every major decision with rationale, alternatives considered, and reversibility assessment |
| Milestone Tracker | Monitors progress against the roadmap, flags delays, and suggests course corrections |
| Risk Radar | Continuously scans for risks (market, technical, financial, personal) and suggests mitigations |

### Layer 3 — Executor Pack (Agents That Ship)

**Purpose:** Execute the actual work of building a startup. These agents don't advise — they produce deliverables.

**Philosophy:** The bottleneck for solo founders isn't knowing what to do — it's doing it. The Executor Pack does the work, producing real artifacts: code, emails, content, financial models, legal documents.

| Agent | Role |
|-------|------|
| Builder Agent | Generates full codebases: landing pages, auth systems, payment integration template, dashboards. Deploys to Vercel/Railway |
| Research Agent | Deep-dives into markets, competitors, technologies, and customer segments. Produces structured research briefs |
| Outreach Agent | Composes and sends personalized email/LinkedIn outreach sequences to potential customers and partners |
| Content & SEO Agent | Writes blog posts, landing page copy, social media content, and optimizes for search engines |
| Sales Agent | Crafts sales scripts, follow-up sequences, objection handlers, and demo scripts |
| Design Agent | Generates UI mockups, brand guidelines, color palettes, and design system recommendations |
| Financial Modeling Agent | Builds revenue projections, burn rate calculators, unit economics models, and pricing optimization |
| Legal & Compliance Agent | Drafts terms of service, privacy policies, cookie policies, and flags regulatory considerations |
| Fundraising Agent | Creates pitch decks, investor outreach lists, financial summaries for fundraising rounds |
| Analytics & Insight Agent | Analyzes product metrics, user behavior, conversion funnels, and produces actionable insights |

### Layer 4 — The Compass (Coaching & Accountability)

**Purpose:** Provide the mentorship, accountability, and emotional support that a solo founder typically gets from a co-founder or advisor.

**Philosophy:** Building alone is psychologically brutal. The Compass is the voice that says "you're making progress" when the founder can't see it, and "you're deluding yourself" when they need to hear it.

| Agent | Role |
|-------|------|
| Daily Standup | Runs a daily check-in: what was done yesterday, what's planned today, blockers. Takes < 3 minutes |
| Weekly Debrief | Weekly review of progress, metrics, learnings, and plan adjustment. Includes momentum analysis |
| Momentum Score | Calculates a real-time score (0–100) based on task completion rate, streak length, revenue progress, and emotional signals |
| Accountability Mode | Sends proactive nudges, deadline reminders, and "you committed to this" messages when the founder goes dark |
| Mirror Agent | The anti-delusion agent. Challenges assumptions, asks hard questions, and flags confirmation bias |
| Founder Knowledge Graph | Maintains a graph of everything the founder has learned: decisions, pivots, lessons, relationships, preferences |
| Mentor Library | A library of synthesized wisdom from startup mentors, founders, and business thinkers, applied contextually |

### Layer 5 — Platform & Infrastructure

**Purpose:** The technical foundation that makes everything work: persistent memory, agent orchestration, integrations, and secure storage.

| Component | Role |
|-----------|------|
| Karnex Memory | Persistent context engine. Stores and retrieves everything the founder has shared, decided, and built — across all sessions |
| Agent Orchestration | Chains agents together. When Research Agent finishes, it hands off to Positioning Agent, then to Builder Agent — automatically |
| Integrations Hub | Manages all third-party connections: GitHub, OxaPay, Gmail, LinkedIn, Notion, PostHog, etc. |
| Founder Vault | Encrypted, secure storage for sensitive documents: contracts, financial data, API keys, passwords |

## Strategic Rationale for payment gateway selections

The transition to OxaPay as the primary payment processor for v1 represents a strategic decision to focus on global accessibility and deployment velocity. Karnex targets a builder-first, tech-forward audience of indie hackers and solo founders who are highly comfortable with cryptocurrency. Utilizing stablecoins (USDT and USDC) removes volatility risk while bypassing Stripe [v2 - deferred]'s geographical constraints, onboarding delays, and compliance overhead. This allows solo founders anywhere in the world to launch and monetize their startup immediately, accelerating time-to-first-value. Traditional billing via Stripe [v2 - deferred] **[v2 - deferred]** is explicitly deferred to v2.

## Non-Goals (What Karnex Does NOT Do)

1. **Karnex is not a general-purpose AI assistant.** It does not answer random questions, write poems, or help with homework.
2. **Karnex does not replace human co-founders.** It augments solo founders — if you have a team, many features are less relevant.
3. **Karnex does not guarantee success.** It reduces the execution gap but cannot fix fundamentally bad ideas.
4. **Karnex does not manage existing large teams.** It's not a project management tool for organizations.
5. **Karnex does not handle DevOps at scale.** Builder Agent deploys simple apps. Complex infrastructure (Kubernetes, multi-region) is out of scope.
6. **Karnex does not provide legal advice.** Legal Agent generates templates, not counsel. Always consult a qualified attorney.
7. **Karnex does not do real-time customer support.** It helps founders, not their end users.
8. **Karnex does not support enterprise sales cycles.** It's optimized for self-serve SaaS, not complex B2B enterprise.
9. **Karnex does not do hardware products.** Software SaaS products only.
10. **Karnex does not integrate with every tool.** We support a curated, opinionated set of integrations.

## Key Product Decisions & Rationale

| Decision | Choice | Rationale |
|----------|--------|-----------|
| **Primary LLM** | Google Gemini | Best price/performance ratio for agent workloads; strong multimodal capabilities; good context windows for complex agent prompts |
| **Agent Runtime** | Google AntiGravity | Purpose-built for agentic execution; native tool-use support; handles long-running agent tasks reliably |
| **Frontend Framework** | Next.js 14 (App Router) | Industry standard for React SSR/SSG; excellent DX; Vercel deployment is seamless; App Router enables server components for performance |
| **Component Library** | shadcn/ui | Not a dependency — copy-paste components that we own and can customize. No version lock-in |
| **Database** | Supabase (PostgreSQL) | Auth, database, storage, and realtime in one service. Perfect for solo dev velocity. RLS provides security without a separate auth layer |
| **Styling** | Tailwind CSS | Utility-first CSS is fastest for solo dev and AI-generated UI code |
| **Agent Orchestration** | LangGraph | State-machine-based agent orchestration with built-in persistence, human-in-the-loop support, and graph-based agent chaining |
| **Backend for Agents** | Python FastAPI | LangChain/LangGraph ecosystem is Python-native; FastAPI provides async performance with minimal boilerplate |
| **Payments (v1)** | OxaPay | Primary processor for v1 launch. Stablecoin-only (USDT/USDC) eliminates volatility. Allows immediate global deployment without Stripe [v2 - deferred]'s geo-restrictions or merchant verification delays. |
| **Payments (v2)** | Stripe [v2 - deferred] | Deferred to v2 as a credit-card alternative billing option. Database structures are processor-agnostic to support Stripe [v2 - deferred] as a drop-in integration. |
| **Email** | Resend | Modern, developer-friendly email API; great deliverability; simple pricing. Critical for mailing OxaPay renewal links. |
| **Analytics** | PostHog | Open-source, self-hostable, full product analytics suite. No sampling. Session recordings |
| **Multi-agent over single-agent** | Multi-agent architecture | Specialized agents outperform generalist agents. Each agent can have optimized prompts, tools, and context. Enables parallel execution |
| **Opinionated coaching** | Mirror Agent + Accountability Mode | Solo founders' biggest risk is self-delusion and momentum loss. Passive tools don't address this. Active coaching does |
| **Credit-based metering** | Agent tasks as credits | Predictable costs for both Karnex and founders. Prevents abuse. Creates natural upgrade triggers |
